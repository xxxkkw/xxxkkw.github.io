团队开发入门指南

面向对象

  • 团队成员（macOS/Windows/Linux），首次连接 GitLab 并开始日常开发。
  • 默认用 SSH 克隆与推送；并且需要先导入根证书（CA）。

一、导入 CA 根证书（用于 HTTPS 和浏览器访问）

若使用 SSH 且不访问 HTTPS，可跳过；但推荐导入，避免浏览器与工具链证书报错。

macOS（图形界面）

  • 双击 internal-root-ca.crt，在“钥匙串访问”选择“系统”或“登录”钥匙串。
  • 展开“信任”，将“使用此证书时”设为“始终信任”。

Windows（图形界面）

  • 双击证书 →“安装证书”→ 选择“本地计算机”→ 存储到“受信任的根证书颁发机构”。

在 GitLab Web 端（服务器地址：https://10.151.30.8）

  • 使用管理员提供的用户名或邮箱登录；如使用临时密码，按提示完成首登。
  • 首次登录请立即修改密码：右上角头像 → Settings → Password（或 Edit profile → Password），输入当前密码并设置强密码。
  • 若浏览器提示证书不受信，请先导入 CA 根证书后再访问。
  • 然后进入头像 → Settings → SSH Keys 粘贴个人公钥，并填写可识别的标题（如 alice-macbook-2025）。

九、团队约定（建议）

  • 统一提交流程与信息格式（如 Conventional Commits）。
  • 禁止强制推送主干（不使用 push --force 到 main）。
  • 推前本地自测；CI 对 main 必跑，红灯立即修复。
  • 未完成功能用特性开关隐藏，避免影响主干可构建性。
  • 不提交敏感数据（PHI/PII），尽量不提交权重，数据等文件。
