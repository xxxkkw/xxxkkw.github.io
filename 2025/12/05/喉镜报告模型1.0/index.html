喉镜报告模型1.0

发表于2025-12-05|更新于2025-12-05
|评论数:

主要思路

模型主要的需求就是输入所有部位的喉镜图片，并且输入受试者的基本情况以及主诉，返回一个结构化的报告，但可实现的路子还是比较多的，特性不同，这里简单谈一下我的想法。

模型视觉编码器预微调

像 MedGemma 或 Qwen3-VL 这种多模态大模型，如果直接进行端到端微调（直接用报告+文本），对于这种高精度病症报告，效果往往不尽人意。

虽然 MedGemma 的视觉部分单图输出 256 个 Token，且我们采用多图拼接的输入方式（每张图独立编码，Token 数量随图片数量线性增加，信息量是足够的），但在直接端到端训练时，模型往往存在“惰性”，视觉 Encoder 倾向于只提取宏观特征（如“这是喉咙”），而忽略细粒度的病灶特征（如“声带前中1/3处的小突起”）。如果视觉 Token 本身没有编码进病灶信息，后续的语言模型再强也无法生成正确的诊断。

因此，拆分训练（两阶段策略）是必要的。我们需要先对视觉部分进行预对齐（Pre-alignment）：

  1. 独立视觉微调：将视觉部分单独拉出来，挂载一个轻量级的 Projector/Adapter语言部分。
  2. 结构化语义对齐：使用构建好的 图片 - [解剖部位, 病灶特征, 异常程度] 结构化数据进行训练。
  3. 强制特征编码：强迫那 256 个 Token 必须包含具体的病灶语义，而不仅仅是通用的图像语义。

通过这种结合式微调，确保视觉部分输出的是富含病理特征的 Token，为后续的报告生成打好基础。

人工合成思维链 (Synthetic CoT)

Nvidia 论文中利用 GPT-OSS-120B 合成思维链（CoT）的思路非常值得借鉴。对于我们而言，既然有现成的金标准报告，完全可以通过逆向工程合成高质量的 CoT 数据。

核心策略：逆向推理（Reverse Reasoning）
单纯的“图片 -> 报告”是黑盒映射，容易产生幻觉。我们需要构建视觉特征 -> 异常归纳 -> 鉴别诊断 -> 最终结论的显式推理路径。

在数据构建时，我们利用大参数量的多模态模型（教师模型），输入图片和最终报告，让其反推：“医生是看到了什么特征（如表面光滑、带蒂、位于声带前中1/3交界处）才下达‘声带息肉’这个诊断的？”。

通过这种方式合成的 CoT 数据，能让小模型学会“如何看图”和“如何推理”，而不仅仅是背诵报告模板。

模型训练 (SFT + RL)

采用 SFT (Supervised Fine-Tuning) -> RL (Reinforcement Learning) 的标准范式。

1. SFT 阶段

利用合成的 CoT 数据和真实报告，让模型学会医学术语规范和报告格式。

2. RL 阶段 (GRPO)

这是提升模型“智商”和准确度的关键。传统的文本相似度（BLEU/ROUGE）在医疗场景意义有限，我们需要设计细粒度的混合奖励函数 (Hybrid Reward)：

  • 诊断准确性 (Hard Reward)：提取生成的诊断结论，与金标准比对，计算 F1 Score 或 Accuracy。诊断正确给予高额正奖励。
  • 格式合规性 (Format Reward)：检查是否符合特定的 JSON 结构或段落模板。
  • 幻觉惩罚 (Negative Constraint)：这是关键。如果模型描述了图中不存在的病灶（无中生有），或者搞错了左右侧，给予严厉的负向奖励。

由于前阶段的视觉预微调已经解决了“看图”的问题，RL 阶段可以更专注于“策略优化”：即如何在准确识别的基础上，生成逻辑严密、措辞精准的报告。
