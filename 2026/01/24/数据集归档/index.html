数据集归档与上传指南

本文档旨在规范内部数据集的归档流程。为了保持 Git 代码仓库的轻量化，我们将代码与原始数据集分离存储。

  • Git 仓库：仅存储数据预处理代码、配置文件和文档。
  • 数据服务器：存储所有原始数据集（Raw Data）和大型中间产物。

📍 目标存储位置

所有数据集将统一归档至内网数据服务器的 /data 目录下。

  • 服务器 IP: 10.108.25.144
  • 根目录: /data

🚀 上传方式指南

我们统一使用 Rsync 进行数据上传，因为它支持断点续传和增量同步，非常适合内网环境。

推荐命令（支持断点续传）

命令模板:

  • -a: 归档模式，保留文件权限和时间戳。
  • -v: 显示详细过程。
  • -P: 显示进度条并支持断点续传。
  • --no-compress: 关键参数。内网传输不要开启压缩，否则会受限于 CPU 性能导致传输变慢。

📝 归档规范 Checklist

在上传前，请确保完成以下检查：

  1. [ ] 不要上传到 Git: 确认 .gitignore 中已经忽略了数据集目录。
  2. [ ] 文件命名: 使用英文、数字、下划线，避免使用空格和中文。
  3. [ ] 文档说明: 在 Git 仓库的 README.md 中说明数据集在服务器上的具体路径，方便其他人调用。

    Example: “原始数据已归档至 10.108.25.144:/data/datasets/nlp/corpus_v1“

  4. [ ] 权限检查: 上传后，建议在服务器上检查一下文件权限，确保团队其他成员有读取权限。
