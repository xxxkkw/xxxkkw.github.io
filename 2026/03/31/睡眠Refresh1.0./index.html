S3Net 下一步可推进方向

方向一：Conditioned Transition Matrix + CRF

核心动机

这条线之所以值得继续做，核心还是因为睡眠分期本身就带有很强的阶段转移结构，尤其是：

  • W、N1、N2 之间的相互转移更频繁
  • N3、REM 更稳定，自环概率更高

但原始转移矩阵本质上还是一个全局静态先验，它能表达平均意义上的转移规律，却表达不了当前样本自己的状态：

  • 当前这个 epoch 的频谱形态更像是即将进入 N2，还是只是处在 W/N1 边界
  • 同样是 N1，前一个时刻来自 W 还是来自 N2，会带来不同的后验转移倾向
  • 同样处在模糊边界段，如果当前有明显 sigma 波 / spindle（纺锤波）趋势，理论上会更偏向 N2；如果 theta 更强，而且 slow rolling eye movement 更明显，那可能更偏向 N1（这里还得回到真实数据里再把各期的统计特征补扎实）

所以一个比较自然的延伸，就是把静态转移矩阵也给揉进故事里面，并且升级成动态版本：

At=Aprior+ΔAt\begin{aligned} A_t &= A_{\text{prior}} + \Delta A_t \end{aligned}At​​=Aprior​+ΔAt​​

其中：

Aprior:由训练集统计得到的全局转移矩阵ΔAt:由当前时刻特征动态生成的 residual transition\begin{aligned} A_{\text{prior}} &: \text{由训练集统计得到的全局转移矩阵} \\ \Delta A_t &: \text{由当前时刻特征动态生成的 residual transition} \end{aligned}Aprior​ΔAt​​:由训练集统计得到的全局转移矩阵:由当前时刻特征动态生成的 residual transition​

最终在 CRF 中使用动态转移矩阵：

S(y1:T)=∑tEt(yt)+∑tAt(yt−1,yt)\begin{aligned} \mathcal{S}(y_{1:T}) &= \sum_t E_t(y_t) + \sum_t A_t(y_{t-1}, y_t) \end{aligned}S(y1:T​)​=t∑​Et​(yt​)+t∑​At​(yt−1​,yt​)​

其中：

Et(yt):分类头输出的 emission logitsAt(yt−1,yt):时刻 t 的条件化转移分数\begin{aligned} E_t(y_t) &: \text{分类头输出的 emission logits} \\ A_t(y_{t-1}, y_t) &: \text{时刻 } t \text{ 的条件化转移分数} \end{aligned}Et​(yt​)At​(yt−1​,yt​)​:分类头输出的 emission logits:时刻 t 的条件化转移分数​

一个偏保守的写法

这里先留一个原则：Conditioned Transition Matrix 只能是微调层，不能反过来主导原模型。

因为当前主干表示已经足够强，所以 transition 分支更适合被当成一个轻量校正项，而不是另起一套以先验为中心的强约束模块。

这里更合适的写法是：

At=f(ht)+λBprior\begin{aligned} A_t &= f(h_t) + \lambda B_{\text{prior}} \end{aligned}At​​=f(ht​)+λBprior​​

其中：

f(ht):由当前时刻特征生成的动态转移矩阵Bprior:由经验转移矩阵得到的弱 bias 项，例如 Bprior=log⁡(Pprior+ϵ) 或其归一化版本ht:时刻 t 的融合特征λ:很小的校准系数，最好设置为较小常数，或者设为可学习标量并初始化到接近 0\begin{aligned} f(h_t) &: \text{由当前时刻特征生成的动态转移矩阵} \\ B_{\text{prior}} &: \text{由经验转移矩阵得到的弱 bias 项，例如 } B_{\text{prior}} = \log(P_{\text{prior}} + \epsilon) \text{ 或其归一化版本} \\ h_t &: \text{时刻 } t \text{ 的融合特征} \\ \lambda &: \text{很小的校准系数，最好设置为较小常数，或者设为可学习标量并初始化到接近 } 0 \end{aligned}f(ht​)Bprior​ht​λ​:由当前时刻特征生成的动态转移矩阵:由经验转移矩阵得到的弱 bias 项，例如 Bprior​=log(Pprior​+ϵ) 或其归一化版本:时刻 t 的融合特征:很小的校准系数，最好设置为较小常数，或者设为可学习标量并初始化到接近 0​

这样写下来，语义上会更稳：

  • 主体仍然由当前特征决定
  • 先验只负责在解码阶段提供一个很轻的方向性校准
  • 不会让 transition 分支因为经验矩阵过强而“压住”原本已经学得很好的表征

如果还想再保守一点，也可以写成：

At=f(ht)+λtBprior,λt∈[0,λmax⁡]\begin{aligned} A_t &= f(h_t) + \lambda_t B_{\text{prior}}, \qquad \lambda_t \in [0, \lambda_{\max}] \end{aligned}At​​=f(ht​)+λt​Bprior​,λt​∈[0,λmax​]​

其中，上界写作：

λmax⁡\begin{aligned} \lambda_{\max} \end{aligned}λmax​​

它取一个很小的值，使先验只能做小幅修正，而不是主导转移结构。

约束项怎么理解

这里的约束也要遵循同一个思路：不是把动态矩阵硬往先验上拉，而是限制“修正幅度别太大”。

如果后面真的做这块，约束也尽量轻一点：

  1. 小幅校正约束
Lcalib=∥At−f(ht)∥F2\begin{aligned} \mathcal{L}_{calib} &= \|A_t - f(h_t)\|_F^2 \end{aligned}Lcalib​​=∥At​−f(ht​)∥F2​​

作用就是限制先验注入后的校正幅度别太大，保证 transition 真的只是“微调”。

  1. 时间平滑约束
Lsmooth=∥At−At−1∥1\begin{aligned} \mathcal{L}_{smooth} &= \|A_t - A_{t-1}\|_1 \end{aligned}Lsmooth​​=∥At​−At−1​∥1​​

作用：防止相邻时刻转移矩阵剧烈跳动。

  1. 结构化轻量约束

不一定直接输出完整 25 个自由参数，也可以考虑：

  • 低秩分解
  • diagonal + off-diagonal bias
  • 按 sleep group 结构化建模

这里的重点也不是强化先验，而是减少参数自由度，避免一个本来只是补充的分支学得过重。

这部分的叙事可以怎么收

这条线可以概括为：

  • Feature-Driven Dynamic Transition 负责主体建模
  • Weak Physiological Bias 负责轻量校准
  • CRF Decoder 保证序列层面的最优解码

这样写会更符合当前模型的定位：原网络还是主角，conditioned transition 只是把序列一致性再往前推一步，而不是额外造一个“先验主导”的系统。

这里还可以顺手定义一个指标：

  • 非法转移率 / 不合理转移率

例如统计预测序列中非常罕见或违反生理常识的阶段跳转比例，用来说明 conditioned transition 确实改善了生理一致性。

方向二：只改门控，不改专家内部

当前判断

这一块我现在更倾向于只动 gate，不动 expert。原因也比较直接：

  • 一个 30s epoch 往往并不只包含单一波形
  • 很多关键波形是短暂事件，不是整个 epoch 的唯一主导结构
  • 真要用“明确属于哪种波”做硬标签，本身边界就很模糊
  • 本身门控最开始做的就很简单，一个交叉注意力 + 分类头

但如果只停留在“按阶段划分专家”这一层，门控又还是太黑箱，生理解释性不够。

所以更合理的折中方式还是：

保留当前 3+2 的专家结构不变，仅改造 gate，使 gate 的决策显式依赖可解释的生理特征。

即：

  • 专家仍然是 Hard group: (W, N1, N2)
  • Easy group: (N3, REM)
  • 专家内部结构不动
  • 只让门控由“融合特征 + 生理描述子”共同决定

改造方式

先约定符号：

ht:当前时刻的高层融合特征pt:生理描述子\begin{aligned} h_t &: \text{当前时刻的高层融合特征} \\ p_t &: \text{生理描述子} \end{aligned}ht​pt​​:当前时刻的高层融合特征:生理描述子​

则门控改成：

wt=softmax(gh(ht)+γgp(pt))\begin{aligned} w_t &= \mathrm{softmax}(g_h(h_t) + \gamma g_p(p_t)) \end{aligned}wt​​=softmax(gh​(ht​)+γgp​(pt​))​

其中：

gh(ht):原始特征驱动的门控分数gp(pt):生理特征驱动的门控偏置γ:控制生理先验对门控的影响强度\begin{aligned} g_h(h_t) &: \text{原始特征驱动的门控分数} \\ g_p(p_t) &: \text{生理特征驱动的门控偏置} \\ \gamma &: \text{控制生理先验对门控的影响强度} \end{aligned}gh​(ht​)gp​(pt​)γ​:原始特征驱动的门控分数:生理特征驱动的门控偏置:控制生理先验对门控的影响强度​

如果想把原始 3+2 的先验逻辑保留得更明显一点，也可以写成：

wt=softmax(gh(ht)+γgp(pt)+bstage-prior)\begin{aligned} w_t &= \mathrm{softmax}(g_h(h_t) + \gamma g_p(p_t) + b_{\text{stage-prior}}) \end{aligned}wt​​=softmax(gh​(ht​)+γgp​(pt​)+bstage-prior​)​

这里的结构偏置项写作：

bstage-prior\begin{aligned} b_{\text{stage-prior}} \end{aligned}bstage-prior​​

它相当于保留现有 sleep-stage grouping 的结构偏置。

这条线到底在改什么

本质上并不是把 SAE 从“stage-aware”改成“wave-aware”；

而是把它升级成：

stage-aware skeleton + physiology-aware gate

也就是：

  • 阶段分组继续负责全局结构先验
  • 波形/频率信息负责局部软调制

这样比直接做“波形专家”更稳，也更容易把方法逻辑讲清楚。

AASM 语义下，各睡眠分期真正对应的生理特征

这里要先记一下：AASM 原始手册本身是标准文档，不是那种完全开放的教材文本；不过公开的临床综述、教学资料和医学综述里，对 AASM 评分标准的描述已经足够支撑模型设计。

下面先把和 AASM 评分实践一致、同时又适合落到模型里的特征顺一遍。

Wake (W)

典型特征：

  • EEG 中后头区 alpha 活动明显，约 8–13 Hz
  • 常伴较高 EMG 活性
  • EOG 可见眨眼或清醒期眼动

如果落到模型上，可以对应成：

  • alpha relative power 高
  • chin EMG 强
  • 低频慢波占比低

N1

典型特征：

  • 清醒向睡眠过渡期
  • 低电压混合频率
  • theta 活动增加，约 4–7 Hz
  • alpha 相比清醒下降
  • 可见 slow rolling eye movements
  • 可见 vertex sharp waves

如果落到模型上，可以对应成：

  • theta/alpha 比值上升
  • alpha power 下降
  • 若有 EOG，则慢眼动强度上升
  • 若能检测 vertex sharpness，可作为辅助特征

N2

典型特征：

  • 背景仍以低电压混合频率为主
  • 出现 sleep spindles
  • 出现 K-complex

这里面最关键的还是：

  • sleep spindle：约 11–16 Hz，常用 12–14 / 12–15 Hz 作为主要关注段
  • K-complex：高幅度双相波，持续时间通常要求足够长，常被视为 N2 标志事件

如果落到模型上，可以对应成：

  • sigma band power 增强
  • spindle density / spindle count 增加
  • K-complex count 或 K-complex score 增加

N3

典型特征：

  • 深睡眠 / slow-wave sleep
  • 高幅度慢波显著
  • delta 主导
  • AASM 常见实践标准：30 秒 epoch 中慢波（通常 0.5–2 Hz、高幅度）占比达到一定阈值时判为 N3

如果落到模型上，可以对应成：

  • delta relative power 显著升高
  • slow-wave 占比升高
  • 低频高幅事件显著

REM

典型特征：

  • 低电压混合频率
  • EEG 相对去同步化，更接近清醒
  • 可见 sawtooth waves
  • 快速眼动明显
  • chin EMG 低张力或接近失张力

如果落到模型上，可以对应成：

  • theta / beta 的混合活性
  • 若有 EOG，REM eye movement 指标增强
  • 若有 EMG，肌张力显著下降

如果不硬定义“某种波”，那门控到底该吃什么

这里更稳妥的做法，不是人工规定“这一定是 spindle 类 expert、delta 类 expert”，而是提取一组软生理描述子给 gate：

pt=[delta,theta,alpha,sigma,beta,event scores,ratios,EOG/EMG cues]\begin{aligned} p_t &= [\text{delta}, \text{theta}, \text{alpha}, \text{sigma}, \text{beta}, \text{event scores}, \text{ratios}, \text{EOG/EMG cues}] \end{aligned}pt​​=[delta,theta,alpha,sigma,beta,event scores,ratios,EOG/EMG cues]​

可以先从下面这些做起：

频段特征

  • delta power: 0.5–4 Hz
  • theta power: 4–8 Hz
  • alpha power: 8–12 Hz
  • sigma power: 12–16 Hz
  • beta power: 16–30 Hz

这里更倾向优先用相对功率，即：

RelPowerb=∑f∈bPSD(f)∑f=fminfmaxPSD(f)\begin{aligned} \mathrm{RelPower}_{b} &= \frac{\sum_{f \in b} PSD(f)}{\sum_{f = f_{min}}^{f_{max}} PSD(f)} \end{aligned}RelPowerb​​=∑f=fmin​fmax​​PSD(f)∑f∈b​PSD(f)​​

比值特征

这些比值很多时候会比单独的 bandpower 更有判别性：

  • theta / alpha
  • sigma / theta
  • delta / beta
  • delta / (alpha + beta)

事件特征

  • spindle density
  • spindle mean amplitude
  • K-complex count / score
  • slow-wave ratio
  • vertex sharp wave score
  • sawtooth-like score

如何从真实数据中统计各分期的频段情况

如果想知道“真实数据里各个 stage 到底是什么频段更强”，不能只靠印象，还是得直接回到标注数据上做统计。

一个比较顺手的统计流程

第一步：按 30s epoch 切分

与标签保持一致，以 epoch 为基本单位。

第二步：做标准预处理

建议至少包括：

  • 去伪迹 / 去坏段
  • bandpass，例如 0.3–35 Hz 或 0.5–40 Hz
  • 必要时 notch filter
  • 重参考或按你当前 pipeline 保持一致

第三步：每个 epoch 估计 PSD

这里可以先用两种常见方案：

  • Welch PSD
  • Multitaper PSD

Welch 更容易上手，也更适合先做大规模统计。

第四步：积分得到 bandpower

对每个 epoch、每个通道、每个频段计算：

Pb=∑f∈bPSD(f)\begin{aligned} P_b &= \sum_{f \in b} PSD(f) \end{aligned}Pb​​=f∈b∑​PSD(f)​

频段先按下面这套来切就够用了：

  • delta: 0.5–4
  • theta: 4–8
  • alpha: 8–12
  • sigma: 12–16
  • beta: 16–30

第五步：统一做相对功率

不同受试者、不同设备、不同夜晚的绝对功率差异会很大，所以这里更适合先看 relative power：

RelPowerb=PbP0.5−30\begin{aligned} \mathrm{RelPower}_b &= \frac{P_b}{P_{0.5-30}} \end{aligned}RelPowerb​​=P0.5−30​Pb​​​

第六步：按 stage 聚合

对所有带标签 epoch 按 W / N1 / N2 / N3 / REM 分组，统计：

  • 均值
  • 标准差
  • 中位数
  • IQR

最后大概会得到：

  • 每个 stage 的 bandpower 分布
  • 每个 stage 的 ratio 分布
  • 每个 stage 的 event 分布

第七步：做显著性和可分性分析

这一步至少可以先做：

  • 各 stage 的箱线图 / violin plot
  • stage 间统计检验
  • 可分性度量

可以重点分析：

  • N1 vs W
  • N1 vs N2
  • N2 vs N3
  • REM vs W

因为这些边界本来就是最难的。

一个更贴近当前项目的门控改造方案

命名

名字上可以先这样想：

  • Physiology-Guided Gate
  • Spectro-Physiological Gate
  • Physiology-Conditioned Stage-Aware Gate

这里面最后一个和当前 SAE 的延续性最好。

核心结构

保持原始两个专家不变：

  • Hard expert: (W, N1, N2)
  • Easy expert: (N3, REM)

只把 gate 的输入从原来的高层表示，扩展成：

Gate Input=[ht ∥ pt]\begin{aligned} \text{Gate Input} &= [h_t \, \| \, p_t] \end{aligned}Gate Input​=[ht​∥pt​]​

其中：

ht:原始高层融合特征pt:生理描述子\begin{aligned} h_t &: \text{原始高层融合特征} \\ p_t &: \text{生理描述子} \end{aligned}ht​pt​​:原始高层融合特征:生理描述子​

最后：

wt=softmax(MLP([ht∥pt]))\begin{aligned} w_t &= \mathrm{softmax}(MLP([h_t \| p_t])) \end{aligned}wt​​=softmax(MLP([ht​∥pt​]))​

或者保守一点：

wt=softmax(MLP(ht)+MLP(pt))\begin{aligned} w_t &= \mathrm{softmax}(MLP(h_t) + MLP(p_t)) \end{aligned}wt​​=softmax(MLP(ht​)+MLP(pt​))​

后者会更容易解释成“生理分支提供的是 bias，而不是直接替代原 gate”。

写法上更自然的表述

这一块如果后面写进论文，没必要说：

  • “我们把专家改造成按波形划分”

更自然的写法是：

  • “我们保留基于睡眠分期可分性的 3+2 专家结构，同时引入生理可解释的频段/事件描述子来调制门控决策，使专家路由不仅依赖抽象深层特征，也依赖显式的睡眠生理证据。”

这句话的好处是，它同时保住了：

  • 你原始工作的理论连续性
  • 生理可解释性
  • 工程可实现性

一个实际可行的推进顺序

第一阶段

先做 Conditioned Transition Matrix + CRF

原因：

  • 跟当前论文衔接最自然
  • 对 N1/N2/W 边界问题帮助最直接
  • 最容易先出结果

第二阶段

再做 Physiology-Conditioned Gate

原因：

  • 不改专家内部，风险更小
  • 可以直接验证“生理特征是否真的改善路由”
  • 解释性显著增强

第三阶段

最后再看是否需要增加更复杂的事件检测器，或者把 EOG/EMG 线索更强地接入 gate。

小结

目前更偏向的路线是：

  • 转移矩阵部分：从静态 transition prior，升级为 conditioned transition prior，并配合 CRF 做序列解码
  • SAE 部分：保留 3+2 专家结构，只改 gate，让 gate 显式依赖频段功率、关键事件和 EOG/EMG 的生理线索

其本质是：

阶段分组负责结构先验，生理特征负责局部调制，CRF 负责序列一致性。

这样走不会推翻当前工作，而且往前推进也比较自然。
